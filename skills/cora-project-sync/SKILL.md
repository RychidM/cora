# Vault Project Sync

Writes per-agent context files into a project repo so any AI coding
agent (Claude Code, Gemini CLI, GitHub Copilot, Codex) loads the same
vault context at session start.

This mirrors `scripts/sync-memory.sh` so the behaviour is available even
when shell scripts can't run (e.g. from Claude desktop).

It also installs per-agent `SessionStart` hooks and a fallback
`MANDATORY SESSION START CONTEXT` block at the top of each generated file
so the Session Start Protocol is enforced even when an agent ignores the
embedded instructions.

---

## Step 1 — Locate the vault

Resolve vault root (see `cora-logger` Step 1): actively read
`$AGENT_MEMORY_VAULT` (run `echo "${AGENT_MEMORY_VAULT:-}"`; the value is not
visible otherwise), then fall back to `list_allowed_directories` (Filesystem
MCP), then `~/obsidian-memory-vault`. Confirm `{vault_root}/AGENTS.md` exists
before proceeding; if the vault can't be located, stop and ask the user rather than
guessing.

---

## Step 2 — Determine target repo

From the conversation:
- If the user names a project (e.g. "sync agentwatch-relay"), look up its repo
  path from `{vault_root}/.project-paths` or ask the user for the path
- If the user is running this from within a repo and the agent has access to
  the current working directory, use that

Determine the project name from the repo's folder name (basename).

---

## Step 3 — Find the matching vault project

Search for a folder matching the project name:
1. `projects/{name}/` — top-level
2. `projects/*/{name}/` — module under a parent

Use the first match.

Note whether the project is a module (parent dir contains its own
`OVERVIEW.md`).

If no match is found, proceed anyway — the synced file will contain only
the global `AGENTS.md` content, with a note that no project-specific
context was attached.

---

## Step 4 — Compute mandatory session-start context

Before assembling each agent file, compute the session-start recap that will
be injected at the top of every generated file and returned by every
per-agent `SessionStart` hook.

The helper script lives at `{vault_root}/scripts/cora-session-start.sh`. If
it is missing, install it first:

- Look for the source copy in the `cora` plugin repo at
  `{cora_repo}/scripts/cora-session-start.sh` or `cora-vault` repo at
  `{cora_vault_repo}/scripts/cora-session-start.sh`.
- If a source copy exists, copy it to `{vault_root}/scripts/cora-session-start.sh`
  and make it executable (or set permissions via Filesystem MCP).
- If no source copy is available, create the helper from the canonical
  implementation in this plugin's source.

Run the helper:

```bash
{vault_root}/scripts/cora-session-start.sh [project_name]
```

- Pass the project name as an argument.
- If the helper emits an error, log it and continue with a short fallback
  note (never fail the whole sync).
- The helper outputs plain text; capture it verbatim.

If the helper is missing and cannot be installed, fall back to a short note
that the session-start context could not be computed.

---

## Step 5 — Build the agent file content

Concatenate, in this order:

```
# MANDATORY SESSION START CONTEXT — surfaced by per-agent SessionStart hook
# If you are reading this file directly, you MUST surface this section to the
# user before answering, unless they opened with a quick one-off question or a
# non-project context. See AGENTS.md Session Start Protocol.
#
{session_start_context}

# Agent Memory
# Auto-generated by cora-project-sync — do not edit manually.
# Vault: {vault_root}
# Synced: {ISO 8601 timestamp}

{contents of AGENTS.md}
```

If a matching project folder was found:

```

---
# Module Context: {project_name}

{contents of {vault_project_dir}/OVERVIEW.md}
```

If the project is a module:

```

---
# System Context: {parent_name} (parent project)

{contents of {parent_dir}/OVERVIEW.md}
```

---

## Step 6 — Write the agent files

Write the assembled content to each of these paths in the repo:

| Agent | Path |
|-------|------|
| Claude Code | `{repo}/.claude/CLAUDE.md` |
| Gemini CLI | `{repo}/.gemini/GEMINI.md` |
| GitHub Copilot | `{repo}/.github/copilot-instructions.md` |
| Codex | `{repo}/.codex/instructions.md` |

Create intermediate directories as needed.

All four targets are tool-specific and safe to overwrite.

---

## Step 7 — Install per-agent SessionStart hooks

For each agent, create a wrapper script and a hook config file that calls
the vault helper and returns the recap in the format that agent expects.

Keep the existing `{repo}/.github/hooks/hooks.json` untouched — user hooks
are allowed to coexist.

### Claude Code

- Wrapper: `{repo}/.claude/hooks/cora-session-start.sh`
- Config: `{repo}/.claude/settings.json` (merge; don't overwrite other hooks)

Wrapper returns:

```json
{
  "hookSpecificOutput": {
    "hookEventName": "SessionStart",
    "additionalContext": "<recap text>"
  }
}
```

### Codex

- Wrapper: `{repo}/.codex/hooks/cora-session-start.sh`
- Config: `{repo}/.codex/hooks.json`

Wrapper returns:

```json
{
  "hookSpecificOutput": {
    "hookEventName": "SessionStart",
    "additionalContext": "<recap text>"
  }
}
```

### Gemini CLI

- Wrapper: `{repo}/.gemini/hooks/cora-session-start.sh`
- Config: `{repo}/.gemini/hooks.json`

Wrapper returns:

```json
{
  "hookSpecificOutput": {
    "additionalContext": "<recap text>"
  }
}
```

### GitHub Copilot CLI

- Wrapper: `{repo}/.github/hooks/cora-session-start.sh`
- Config: `{repo}/.github/hooks/cora-session-start.json`
- Leave `{repo}/.github/hooks/hooks.json` untouched.

Wrapper returns:

```json
{
  "additionalContext": "<recap text>"
}
```

Hook config merging rules:
- If the config file does not exist, create it with only the `SessionStart`
  hook.
- If it exists and already has a `SessionStart` hook, prepend or replace
  ours (identified by a `cora-session-start` marker) and preserve other
  hooks.
- If it exists but has no `hooks.SessionStart` block, add it.
- Use `jq` when available; otherwise warn and print the manual merge
  snippet.

---

## Step 8 — Ensure .gitignore covers the generated files

Read `{repo}/.gitignore`. If it doesn't already contain the agent-files
block, append:

```

# Agent memory files — auto-generated by cora-project-sync, do not commit
.claude/
.gemini/
.codex/
.github/copilot-instructions.md
.github/hooks/cora-session-start*
```

If the `.gitignore` doesn't exist, create it with just this block.

If the repo still contains the old root-level files (`CLAUDE.md`,
`GEMINI.md`, `AGENTS.md`) and they were auto-generated by this skill,
safely remove them after writing the new per-directory files. If they were
hand-authored, leave them untouched and warn the user.

```
Note: no vault folder found for '{name}' — synced AGENTS.md only.
Run /cora-init {name} {repo_path} to register the project.
```

---

## Rules

- **Never commit the generated files** — they're auto-generated and
  always gitignored
- **Always include the timestamp** — so the user knows when content was last
  refreshed
- **Don't modify AGENTS.md or any vault file** — this skill is one-way
  (vault → repo)
- **Handle paths with spaces correctly** — the local Mac vault path
  contains spaces; quote everything
- **Leave existing user hooks untouched** — CORA hooks are additive
